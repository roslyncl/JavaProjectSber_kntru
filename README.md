# Backend API для управления клиентами и товарами

REST API сервис на Spring Boot для системы управления клиентами, товарами, поставщиками и адресами. Учебный проект, демонстрирующий навыки backend-разработки на Java.

## Технологии

- **Java 17+**
- **Spring Boot**
- **Spring Data JPA** (для работы с базой данных)
- **PostgreSQL** (реляционная база данных)
- **SpringDoc OpenAPI** (Swagger UI для документации API)
- **Maven** (система сборки)
- **Lombok** (упрощение кода)
- **Jakarta Validation** (валидация входных данных)

## Документация API

Проект использует **SpringDoc OpenAPI** для автоматической генерации документации.

После запуска приложения документация доступна по адресам:

- **Swagger UI:** http://localhost:8081/swagger-ui/index.html
- **OpenAPI JSON:** http://localhost:8080/v3/api-docs

## Основные сущности

1. **Address** (Адрес) - хранит информацию об адресах (улица, город, страна)
2. **Client** (Клиент) - данные клиентов с привязкой к адресу (имя, фамилия, дата рождения)
3. **Product** (Товар) - информация о товарах с валидацией цены и остатков
4. **Supplier** (Поставщик) - данные поставщиков с контактами

## Связи между сущностями

- **Client** ↔ **Address** - One-to-One (один клиент имеет один адрес)
- **Supplier** ↔ **Address** - One-to-One (один поставщик имеет один адрес)
- **Product** ↔ **Supplier** - Many-to-One (много товаров у одного поставщика)

## API Endpoints

### Клиенты (`/api/clients`)
- `GET /` - получить всех клиентов
- `GET /{id}` - получить клиента по ID
- `POST /` - создать нового клиента
- `PUT /{id}` - обновить клиента
- `DELETE /{id}` - удалить клиента
- `GET /by-city/{city}` - найти клиентов по городу (нативный SQL запрос)

### Товары (`/api/products`)
- `GET /` - получить все товары
- `GET /{id}` - получить товар по ID
- `POST /` - создать новый товар
- `PUT /{id}` - обновить товар
- `DELETE /{id}` - удалить товар
- `GET /by-supplier-name/{supplierName}` - найти товары по имени поставщика (нативный SQL запрос)

### Адреса (`/api/address`)
- `GET /` - получить все адреса
- `GET /{id}` - получить адрес по ID
- `POST /` - создать новый адрес
- `PUT /{id}` - обновить адрес
- `DELETE /{id}` - удалить адрес

### Поставщики (`/api/supplier`)
- `GET /` - получить всех поставщиков
- `GET /{id}` - получить поставщика по ID
- `POST /` - создать нового поставщика
- `PUT /{id}` - обновить поставщика
- `DELETE /{id}` - удалить поставщика

## Что демонстрирует проект

- Слоистую архитектуру Spring Boot (Controller-Service-Repository)
- Работу с JPA и реляционными связями между сущностями
- REST API с полным набором CRUD-операций
- Валидацию данных с использованием Jakarta Validation
- Логирование операций с помощью Slf4j
- Обработку исключений через кастомные исключения
- Документирование API с использованием Swagger/OpenAPI
- Нативные SQL-запросы для сложных выборок данных
